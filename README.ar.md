# OpenWrt Smart Reboot (مع LuCI)

- الإنجليزية: [README.en.md](README.en.md)
- فهرس متعدد اللغات: [README.i18n.md](README.i18n.md)
- دليل الإرسال إلى OpenWrt upstream: [OPENWRT_UPSTREAM_SUBMISSION.md](OPENWRT_UPSTREAM_SUBMISSION.md)

تقوم هذه الحزمة بإعادة تشغيل الموجّه في وقت الفجر المحدد فقط عندما تكون الشبكة في حالة خمول.

## المكونات

- `smart-reboot`: خدمة خلفية لفحص الخمول وتنفيذ إعادة التشغيل
- `luci-app-smart-reboot`: صفحة إعداد LuCI

## آلية العمل

1. تشغيل `smart-reboot-check` في الوقت المحدد (الافتراضي: `04:00`)
2. قياس `rx_bytes + tx_bytes` على الواجهات المحددة لمدة `sample_seconds`
3. إذا كانت الزيادة ≤ `byte_threshold` يتم اعتبار الشبكة خاملة وتنفيذ إعادة التشغيل

## القيم الافتراضية

- مفعّل: `1`
- الوقت: `04:00`
- اختيار كل الواجهات: `0`
- الواجهة: `wan`
- مدة العينة: `120`
- حد الخمول: `262144` بايت (256KB)

## خيارات LuCI

- وقت إعادة التشغيل: `HH:MM`
- اختيار كل الواجهات
- الواجهات المراقبة (تسمية منطقية + اسم الجهاز)
- وقت آخر إعادة تشغيل تلقائية
