# OpenWrt Smart Reboot (รองรับ LuCI)

- ภาษาอังกฤษ: [README.en.md](README.en.md)
- ดัชนีหลายภาษา: [README.i18n.md](README.i18n.md)
- คู่มือส่งขึ้น OpenWrt upstream: [OPENWRT_UPSTREAM_SUBMISSION.md](OPENWRT_UPSTREAM_SUBMISSION.md)

แพ็กเกจนี้จะรีบูตเราเตอร์ตามเวลาช่วงเช้ามืดที่กำหนด เฉพาะเมื่อเครือข่ายอยู่ในสถานะว่างเท่านั้น

## องค์ประกอบ

- `smart-reboot`: บริการ backend สำหรับตรวจสถานะว่างและสั่งรีบูต
- `luci-app-smart-reboot`: หน้าตั้งค่า LuCI

## การทำงาน

1. เรียก `smart-reboot-check` ตามเวลาที่ตั้งไว้ (ค่าเริ่มต้น: `04:00`)
2. วัดค่า `rx_bytes + tx_bytes` ของอินเทอร์เฟซที่เลือกเป็นเวลา `sample_seconds`
3. หากปริมาณเพิ่มขึ้น ≤ `byte_threshold` จะถือว่าเครือข่ายว่างและรีบูต

## ค่าเริ่มต้น

- เปิดใช้งาน: `1`
- เวลา: `04:00`
- เลือกทุกอินเทอร์เฟซ: `0`
- อินเทอร์เฟซ: `wan`
- เวลาสุ่มตัวอย่าง: `120`
- เกณฑ์ว่าง: `262144` ไบต์ (256KB)

## ตัวเลือก LuCI

- เวลารีบูต: `HH:MM`
- เลือกทุกอินเทอร์เฟซ
- อินเทอร์เฟซที่เฝ้าตรวจ (ป้ายกำกับเชิงตรรกะ + ชื่ออุปกรณ์)
- เวลาการรีบูตอัตโนมัติล่าสุด
